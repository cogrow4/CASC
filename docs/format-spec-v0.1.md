# CASC — Community Audio Source Container
### `.casc` Format Specification v0.1 (Draft)

---

## Overview

CASC (`.casc`) is an open, DAW-agnostic audio plugin format designed around three principles:

1. **One file, all platforms.** A `.casc` file is identical on Windows, macOS, and Linux. No per-platform builds, no installer, no registry.
2. **Native-grade performance.** DSP is compiled to WebAssembly with SIMD support, then AOT-compiled to native machine code at install time.
3. **Safe by default.** Each plugin runs in a memory-isolated sandbox. A crashing plugin cannot corrupt the host.

CASC bridges into the existing plugin ecosystem exclusively via **CLAP** — the only major plugin standard that is fully open source (MIT licensed) with no corporate licensing requirements. There is no VST3 bridge and none is planned. This is a deliberate choice: CASC's community-first values are incompatible with proprietary format dependencies.

---

## 1. File Format

A `.casc` file is a **ZIP archive** with a mandatory internal layout. Rename any `.casc` to `.zip` and it opens normally.

### 1.1 Required directory structure

```
myplugin.casc
├── manifest.json          # Required. Plugin identity, I/O, params.
├── dsp.wasm               # Required. DSP logic. Wasm MVP + SIMD proposal.
└── presets/               # Optional. Factory presets as .json files.
    └── default.json
```

### 1.2 Optional entries

```
├── ui.wasm                # Optional. GUI logic (compiled from C++/Rust/etc.)
├── ui.html                # Optional. HTML/JS GUI (sandboxed WebView)
├── dsp.wasm.aot.x86_64    # Cache: AOT-compiled native slice, x86_64
├── dsp.wasm.aot.arm64     # Cache: AOT-compiled native slice, arm64
└── assets/                # Optional. Fonts, images, IR files, wavetables.
    └── reverb_ir.wav
```

AOT slices (`.aot.*`) are **never shipped by the plugin author**. They are generated by the host runtime on first load and cached inside the file (or alongside it in a platform cache dir, if the file is read-only). The `.wasm` source remains the canonical artifact.

### 1.3 Archive requirements

- Format: ZIP (DEFLATE or STORE, no ZIP64 required for files < 4 GB)
- Encoding: UTF-8 filenames only
- No encryption, no password protection
- The archive MUST NOT contain absolute paths or `../` traversal sequences

## 2. manifest.json

The manifest is the plugin's complete public contract. Hosts parse only this file before deciding whether to load `dsp.wasm`.

### 2.1 Full schema

```json
{
  "casc_version": "0.1",

  "id": "com.yourname.myreverb",
  "name": "My Reverb",
  "version": "1.2.0",
  "vendor": "Your Name",
  "url": "https://yourname.com/myreverb",
  "description": "A lush algorithmic reverb.",
  "category": "reverb",

  "features": ["audio-effect", "stereo", "tail"],

  "audio_inputs": [
    { "name": "main", "channels": 2 }
  ],
  "audio_outputs": [
    { "name": "main", "channels": 2 }
  ],

  "midi_input": false,
  "midi_output": false,
  "mpe_support": false,
  "note_ports": [],

  "latency_frames": 0,
  "tail_seconds": 4.0,
  "hard_realtime": true,

  "params": [
    {
      "id": 0,
      "name": "Room Size",
      "short_name": "Size",
      "module": "Reverb",
      "min": 0.0,
      "max": 1.0,
      "default": 0.5,
      "unit": "",
      "flags": ["automatable", "modulatable"],
      "steps": null
    },
    {
      "id": 1,
      "name": "Damping",
      "short_name": "Damp",
      "module": "Reverb",
      "min": 0.0,
      "max": 1.0,
      "default": 0.3,
      "unit": "",
      "flags": ["automatable", "modulatable"],
      "steps": null
    },
    {
      "id": 2,
      "name": "Mix",
      "short_name": "Mix",
      "module": "Output",
      "min": 0.0,
      "max": 1.0,
      "default": 0.5,
      "unit": "%",
      "flags": ["automatable", "modulatable"],
      "steps": null
    }
  ],

  "gui": {
    "type": "wasm",
    "entry": "ui.wasm",
    "width": 600,
    "height": 400,
    "resizable": false
  }
}
```

### 2.2 Field reference

| Field | Type | Required | Notes |
|---|---|---|---|
| `casc_version` | string | yes | Format version this plugin targets |
| `id` | string | yes | Reverse-DNS unique ID. Stable forever. |
| `name` | string | yes | Display name, ≤ 64 chars |
| `version` | string | yes | SemVer |
| `vendor` | string | yes | Author / company name |
| `category` | string | yes | See §2.3 |
| `features` | string[] | yes | See §2.4 |
| `audio_inputs` | Port[] | yes | Can be empty array for MIDI-only plugins |
| `audio_outputs` | Port[] | yes | Can be empty array |
| `latency_frames` | int | yes | Report 0 if none. Used for PDC. |
| `tail_seconds` | float | yes | Report 0.0 if no tail. |
| `hard_realtime` | bool | yes | Set `true` if `dsp_process()` must never block |
| `params` | Param[] | yes | Can be empty array |
| `gui` | GUI or null | no | Omit if no GUI |

### 2.3 Category values

`analyzer`, `chorus`, `compressor`, `delay`, `distortion`, `drum-machine`, `eq`, `filter`, `flanger`, `generator`, `limiter`, `modulator`, `phaser`, `pitch-shifter`, `reverb`, `sampler`, `sequencer`, `synth`, `transient-shaper`, `utility`, `other`

### 2.4 Feature flags

`audio-effect`, `instrument`, `midi-effect`, `stereo`, `mono`, `surround`, `ambisonic`, `tail`, `hard-realtime`, `requires-internet`

### 2.5 Param flags

`automatable` — host may record/play back value over time  
`modulatable` — host may modulate value per-sample  
`hidden` — not shown in generic host param list  
`readonly` — host cannot set this param (metering use)  
`stepped` — use integer steps; set `steps` to the count  
`bypass` — this param is the plugin's bypass switch

## 3. dsp.wasm — The DSP module

### 3.1 Wasm requirements

- Target: **Wasm MVP** + **Wasm SIMD** (128-bit, `simd128` proposal)
- Memory: one linear memory, exported as `memory`
- Threading: not permitted in v0.1 (single audio thread assumed)
- Floating point: `f32` for audio samples, `f64` permitted for internal state

### 3.2 Exported functions (required)

The host calls these C-compatible exported Wasm functions. All names are exact.

```c
// Called once at startup. Returns an opaque handle.
// sample_rate: e.g. 44100, 48000, 96000
// max_block_size: maximum frames per process() call
i32 dsp_create(f64 sample_rate, i32 max_block_size);

// Called once before destroy. Frees resources.
void dsp_destroy(i32 handle);

// Called whenever sample rate or block size changes.
void dsp_reset(i32 handle, f64 sample_rate, i32 max_block_size);

// Set a parameter by its manifest ID.
// value is normalised 0.0–1.0; host converts from display value.
void dsp_set_param(i32 handle, i32 param_id, f64 value);

// Get current parameter value (normalised).
f64 dsp_get_param(i32 handle, i32 param_id);

// Main audio processing function.
// in_ptr / out_ptr: pointers into Wasm linear memory
// Layout: interleaved float32 channels × frames
// e.g. stereo: [L0, R0, L1, R1, ... L(n-1), R(n-1)]
void dsp_process(
    i32 handle,
    i32 in_ptr,   // float32* in Wasm memory
    i32 out_ptr,  // float32* in Wasm memory
    i32 frames,
    i32 in_channels,
    i32 out_channels
);

// Serialise state to Wasm memory. Returns byte length.
// ptr: pointer to buffer host allocated in Wasm memory
i32 dsp_save_state(i32 handle, i32 ptr, i32 max_bytes);

// Restore state from bytes previously returned by dsp_save_state.
void dsp_load_state(i32 handle, i32 ptr, i32 byte_count);

// Returns latency in frames (may change after reset).
i32 dsp_get_latency(i32 handle);

// Returns tail in frames (may change during processing).
i32 dsp_get_tail(i32 handle);
```

### 3.3 Memory layout contract

The host allocates input and output buffers **inside Wasm linear memory** using exported helper functions:

```c
// Allocate n bytes in Wasm memory. Returns pointer.
i32 casc_alloc(i32 n_bytes);

// Free a pointer previously returned by casc_alloc.
void casc_free(i32 ptr);
```

This keeps all audio data inside the sandbox — no host-memory pointers are ever passed into the Wasm module.

### 3.4 MIDI events (optional, v0.1 preview)

If `midi_input: true` in manifest, the following export is also required:

```c
// Called before dsp_process() for each block.
// events_ptr: pointer to packed MidiEvent array in Wasm memory
// Each MidiEvent: { i32 frame_offset, u8 status, u8 data1, u8 data2, u8 pad }
void dsp_send_midi(i32 handle, i32 events_ptr, i32 event_count);
```

### 3.5 Threading rules

`dsp_process()` is called from the host's **audio thread**. All other functions may be called from any thread but will be serialised by the host. The Wasm module MUST NOT:

- Allocate memory from the system allocator inside `dsp_process()`
- Block on any synchronisation primitive
- Call any Wasm import that could block (file I/O, network)

## 4. AOT Compilation

### 4.1 The flow

```
First load:
  1. Host extracts dsp.wasm from archive
  2. libcasc calls wasmtime compile (or wasm2c + clang)
  3. Native binary written to cache
  4. Future loads skip compilation entirely

Cache location (in order of preference):
  a. Inside the .casc archive itself (if writable)
  b. $XDG_CACHE_HOME/casc/<plugin-id>/ (Linux)
  c. ~/Library/Caches/casc/<plugin-id>/ (macOS)
  d. %LOCALAPPDATA%\casc\<plugin-id>\ (Windows)
```

### 4.2 Cache invalidation

Cache is invalidated (and recompiled) if:
- SHA-256 of `dsp.wasm` does not match stored hash
- Host runtime version changed (different wasmtime ABI)
- CPU feature flags changed (e.g. moved to machine with AVX-512)

### 4.3 Compilation backends

| Backend | Notes |
|---|---|
| Wasmtime (Cranelift) | Recommended. Fast compile, good codegen, ~3% slower than clang -O2 |
| wasm2c + clang -O3 | Best perf ceiling. Slower compile. Good for shipping pre-compiled slices. |
| wasm3 (interpreter) | Fallback only — for platforms where AOT is not available |

## 5. libcasc — Host Runtime

`libcasc` is a small, dependency-light C library that DAW developers embed. It handles all of the above so host authors never touch Wasm directly.

### 5.1 Public API (libcasc.h)

```c
// --- Loading ---

// Load a .casc file. Returns NULL on error.
// err_buf receives a human-readable error message.
casc_plugin_t* casc_load(const char* path, char* err_buf, size_t err_buf_len);

// Unload and free all resources.
void casc_unload(casc_plugin_t* plugin);

// Read the manifest without fully loading the plugin. Fast.
// Returns heap-allocated JSON string. Caller frees.
char* casc_read_manifest(const char* path);

// --- Lifecycle ---

casc_instance_t* casc_instantiate(
    casc_plugin_t* plugin,
    double sample_rate,
    int max_block_size
);

void casc_destroy_instance(casc_instance_t* inst);

void casc_reset(casc_instance_t* inst, double sample_rate, int max_block_size);

// --- Processing ---

void casc_process(
    casc_instance_t* inst,
    const float** inputs,    // [channel][frame]
    float** outputs,         // [channel][frame]
    int frames
);

void casc_send_midi(
    casc_instance_t* inst,
    const casc_midi_event_t* events,
    int count
);

// --- Parameters ---

void casc_set_param(casc_instance_t* inst, int param_id, double value);
double casc_get_param(casc_instance_t* inst, int param_id);

int casc_get_latency(casc_instance_t* inst);   // frames
int casc_get_tail(casc_instance_t* inst);       // frames

// --- State ---

// Returns heap-allocated blob. Caller frees. Sets *out_size.
void* casc_save_state(casc_instance_t* inst, size_t* out_size);
int   casc_load_state(casc_instance_t* inst, const void* data, size_t size);

// --- GUI ---

// Open the plugin GUI. parent_handle is platform native (HWND/NSView/XID).
// Returns 0 on success.
int  casc_open_ui(casc_instance_t* inst, void* parent_handle);
void casc_close_ui(casc_instance_t* inst);
void casc_ui_tick(casc_instance_t* inst);  // Call from UI thread ~60hz
```

### 5.2 Build targets

| Platform | Library | Compiler |
|---|---|---|
| Windows | `libcasc.lib` (static) | MSVC 2019+ or clang-cl |
| macOS | `libcasc.a` (static) | Apple Clang 14+ |
| Linux | `libcasc.a` (static) | GCC 11+ or Clang 14+ |

Single header distribution (`libcasc.h`) with implementations split across `casc_loader.c`, `casc_wasm.c`, `casc_aot.c`, `casc_ui.c`.

Dependencies: **miniz** (ZIP), **wasmtime-c-api** (Wasm), **miniaudio** (optional, for standalone mode only).

## 6. CLAP Bridge

The CLAP bridge (`casc-clap-bridge`) is a standard CLAP plugin (`.clap` file) that acts as an adapter. It hosts any `.casc` file and exposes it to any CLAP-compatible DAW as if it were a native CLAP plugin.

### 6.1 How it works

```
DAW (CLAP host)
  └── casc-clap-bridge.clap
        └── libcasc
              └── myplugin.casc
                    └── dsp.wasm (AOT-compiled)
```

The bridge is **one binary per platform** that can wrap **any number of `.casc` plugins**. DAWs scan the bridge file once; the bridge then reports all installed `.casc` plugins as if they were individual CLAP plugins.

### 6.2 Plugin discovery

The bridge scans these directories for `.casc` files on startup:

```
Windows:  %COMMONPROGRAMFILES%\CASC\
          %APPDATA%\CASC\
macOS:    /Library/Audio/Plug-Ins/CASC/
          ~/Library/Audio/Plug-Ins/CASC/
Linux:    /usr/lib/casc/
          ~/.local/lib/casc/
          $CASC_PATH (colon-separated, like PATH)
```

### 6.3 CLAP feature mapping

| CASC manifest field | CLAP equivalent |
|---|---|
| `id` | `clap_plugin_descriptor.id` |
| `name` | `clap_plugin_descriptor.name` |
| `vendor` | `clap_plugin_descriptor.vendor` |
| `version` | `clap_plugin_descriptor.version` |
| `category` | `clap_plugin_descriptor.features[]` |
| `params[].id` | `clap_param_info.id` |
| `params[].flags` | `clap_param_info.flags` |
| `latency_frames` | `clap_plugin_latency.get()` |
| `tail_seconds` | `clap_plugin_tail.get()` |
| `audio_inputs/outputs` | `clap_plugin_audio_ports` |
| `midi_input` | `clap_plugin_note_ports` |

### 6.4 Parameter automation

CLAP sends parameter changes as events in the process block. The bridge translates these to `casc_set_param()` calls before calling `casc_process()`. This preserves sample-accurate automation.

### 6.5 GUI integration

- If `ui.wasm` or `ui.html` is present, the bridge exposes `CLAP_EXT_GUI`
- `ui.html` GUIs are hosted in a sandboxed `WebView2` (Win), `WKWebView` (Mac), or `WebKitGTK` (Linux) window
- Communication between UI and DSP is via a simple message queue (no shared memory across the sandbox boundary)

### 6.6 State save/restore

CLAP calls `clap_plugin_state.save()` → bridge calls `casc_save_state()` → raw bytes returned to DAW. On load, reverse. The state blob is opaque to the bridge — the plugin owns its format.

## 7. Preset format

Factory presets live in `presets/*.json`. User presets are stored by the host, not inside the file.

```json
{
  "casc_preset_version": "0.1",
  "plugin_id": "com.yourname.myreverb",
  "plugin_version": "1.0.0",
  "name": "Large Hall",
  "author": "Your Name",
  "params": {
    "0": 0.85,
    "1": 0.4,
    "2": 0.6
  }
}
```

Param values are normalised (0.0–1.0). The preset format is intentionally simple — no binary, no compression, human-readable and diffable.

## 8. Security model

- All DSP runs inside a Wasm linear memory sandbox. Plugin cannot read host memory.
- `dsp.wasm` may only import functions from the `casc_host` namespace (provided by libcasc).
- Permitted imports: `casc_host.log(ptr, len)`, `casc_host.get_time_seconds() → f64`
- Forbidden: any file I/O, network, threading, or OS syscalls from within `dsp.wasm`
- GUI code runs in a separate sandboxed process (WebView); it communicates with DSP only through the libcasc message queue

---

## 9. Versioning and compatibility

- `casc_version` in the manifest is the **format version**, not the plugin version
- Hosts MUST reject plugins with a `casc_version` major number higher than they support
- A host supporting `"0.x"` MUST support all `"0.x"` plugins (minor = backwards compatible additions)
- Breaking changes increment the major version
- The Wasm ABI (exported function names) is **frozen** at v1.0. Functions may be added but never removed or renamed after v1.0.

## Appendix A — Minimal working plugin in C (skeleton)

```c
#include <stdint.h>
#include <string.h>
#include <math.h>

// State per instance
typedef struct {
    float param_gain;
    double sample_rate;
} GainPlugin;

static GainPlugin instances[16];  // simple static pool

__attribute__((export_name("dsp_create")))
int32_t dsp_create(double sample_rate, int32_t max_block_size) {
    static int next = 0;
    int h = next++;
    instances[h].sample_rate = sample_rate;
    instances[h].param_gain  = 1.0f;
    return h;
}

__attribute__((export_name("dsp_destroy")))
void dsp_destroy(int32_t h) {}

__attribute__((export_name("dsp_reset")))
void dsp_reset(int32_t h, double sr, int32_t bs) {
    instances[h].sample_rate = sr;
}

__attribute__((export_name("dsp_set_param")))
void dsp_set_param(int32_t h, int32_t id, double v) {
    if (id == 0) instances[h].param_gain = (float)v;
}

__attribute__((export_name("dsp_get_param")))
double dsp_get_param(int32_t h, int32_t id) {
    return (id == 0) ? instances[h].param_gain : 0.0;
}

__attribute__((export_name("dsp_process")))
void dsp_process(int32_t h, int32_t in_ptr, int32_t out_ptr,
                 int32_t frames, int32_t in_ch, int32_t out_ch) {
    float* in  = (float*)in_ptr;
    float* out = (float*)out_ptr;
    float  g   = instances[h].param_gain;
    for (int i = 0; i < frames * out_ch; i++)
        out[i] = in[i] * g;
}

__attribute__((export_name("dsp_get_latency")))
int32_t dsp_get_latency(int32_t h) { return 0; }

__attribute__((export_name("dsp_get_tail")))
int32_t dsp_get_tail(int32_t h) { return 0; }

__attribute__((export_name("dsp_save_state")))
int32_t dsp_save_state(int32_t h, int32_t ptr, int32_t max) {
    memcpy((void*)ptr, &instances[h].param_gain, sizeof(float));
    return sizeof(float);
}

__attribute__((export_name("dsp_load_state")))
void dsp_load_state(int32_t h, int32_t ptr, int32_t n) {
    memcpy(&instances[h].param_gain, (void*)ptr, sizeof(float));
}

// Memory helpers
static uint8_t heap[1 << 20];  // 1 MB static heap
static int heap_top = 0;

__attribute__((export_name("casc_alloc")))
int32_t casc_alloc(int32_t n) {
    int p = heap_top;
    heap_top += (n + 7) & ~7;  // 8-byte align
    return p + (int32_t)heap;
}

__attribute__((export_name("casc_free")))
void casc_free(int32_t ptr) {}  // bump allocator — no free needed for DSP buffers
```

Compile with:
```sh
clang --target=wasm32 -O3 -msimd128 \
      -nostdlib -Wl,--no-entry \
      -Wl,--export-dynamic \
      gain.c -o dsp.wasm
```
